# Learnings - April 2026

## 2026-04-04 | ACC | Score: 16 | CLAUDE.md spec diverges significantly from actual user stories

**What happened:** User stories in `docs/user-story/user-stories.md` reveal requirements far broader than CLAUDE.md captures. Actual needs include: task-level tracking (not just daily progress %), survey phase before execution, evidence uploads (photos/docs/screenshots), Kanban/table views for technicians, manager budget planning, per-task technician assignment with auto-assign, client management (name/address/phone), and project value in rupiah. Current DB schema (`daily_reports` with `progress_percentage`) and SPI calculator are built against the simpler CLAUDE.md model.
**Why it matters:** Every feature built against CLAUDE.md will need rework once actual user stories are implemented. Data model mismatch is the most expensive kind of technical debt.
**Action for next time:** Before any implementation phase, reconcile CLAUDE.md spec with user stories document. Update data model to match actual requirements FIRST. Flag any spec-vs-stories conflicts to user immediately.
**Files involved:** `CLAUDE.md`, `docs/user-story/user-stories.md`, `server/database/schema.sql`, `server/src/services/spiCalculator.ts`

## 2026-04-04 | FIX | Score: 9 | LandingPage.tsx at 870 lines exceeds split threshold

**What happened:** `frontend/src/pages/LandingPage.tsx` is 870 lines -- a single file containing inline SVG icon components and all page sections (hero, IoT info, services, gallery, testimonials, contact, footer). Exceeds the 800-line mandatory split threshold.
**Why it matters:** Large single-file components become maintenance bottlenecks. Pattern could repeat with dashboard pages which will be more complex.
**Action for next time:** Extract SVG icons to a shared `components/icons/` directory. Split page sections into individual components under `components/landing/`. Apply before the file grows further.
**Files involved:** `frontend/src/pages/LandingPage.tsx`
