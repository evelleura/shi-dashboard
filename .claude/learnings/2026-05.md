## 2026-05-05 | RSH | Score: 17 | Indonesian academic SD standard: Raharja business-process pattern

**What happened:** Built sequence diagrams using MVC/technical pattern (User|Sistem|Database). User provided Raharja University KKP reference (widuri.raharja.info KP1122469850). Pattern is fundamentally different: lifelines are BUSINESS OBJECTS (Form Login, Data Pengguna, Dashboard) not technical layers. Multiple actors allowed. Narrative paragraph required after each diagram: "Berdasarkan gambar X.X ... terdapat: 1). N Lifeline ... 2). M Aktor ... 3). K Message ...".
**Why it matters:** Indonesian academic thesis (Tugas Akhir, KKP, skripsi) uses business-process UML, not MVC. Technical layer naming (Controller, Service, Repository) is wrong in this context. Raharja standard is the institutional benchmark.
**Action for next time:** For any Indonesian academic thesis diagram: fetch a Raharja KKP or similar reference FIRST. Apply business-object lifelines (Form/Data/Dokumen), plain role names (Pengguna/Manajer/Teknisi), and include Raharja narrative paragraph format.
**Files involved:** `diagram/STANDARD.md`, `diagram/ai/Sequence/_gen.py`, `naskah/10_sequence_diagram.md`

## 2026-05-05 | FIX | Score: 14 | drawio activation bar clipping: arrows must start at bar edge, not center

**What happened:** Activation bars (white/blue rectangles on lifelines) caused visual clipping when message arrows originated from lifeline center. Cross-lifeline arrows appeared to "pierce" into activation bars. Self-message loops had both waypoints starting inside the bar. Fix: shift source/target x by ±BAR_HALF (bar half-width = 5px) in the direction of travel. Self-msg starts at `x + BAR_HALF` (right edge).
**Why it matters:** Any drawio sequence diagram with activation bars will have this clipping issue unless explicitly offset. Standard UML tools (Visio, StarUML, Papyrus) handle this automatically; drawio XML-level generators do not.
**Action for next time:** In drawio XML generator with activation bars: `msg_cell` shifts x1/x2 by ±BAR_HALF based on direction. `self_msg_cell` starts at `x + BAR_HALF`. Keep `BAR_HALF` as named constant.
**Files involved:** `diagram/ai/Sequence/_gen.py:68-98` (msg_cell, self_msg_cell functions)

## 2026-05-05 | FIX | Score: 15 | Academic diagram iteration cost: no reference = 6+ correction cycles

**What happened:** Generated 4 sequence diagrams across 6+ iterations before reaching acceptable quality. Each iteration user corrected a different aspect: lifeline pattern, icon type (actor vs rectangle), color scheme, activation bars, clipping. Root cause: started generating from spec alone without fetching an actual academic reference example from the target institution.
**Why it matters:** Academic UML diagrams have highly specific institutional conventions (Raharja = business-process lifelines, colored scheme, narrative paragraphs). These conventions are invisible without seeing a reference. Generating without reference = guaranteed rework.
**Action for next time:** Before generating ANY academic diagram (thesis, KKP, skripsi): (1) Ask user for reference paper OR search Widuri/similar repository. (2) Fetch and analyze visual + structural conventions. (3) Generate only after pattern is confirmed. Rule: reference-first, generate-second.
**Files involved:** `diagram/ai/Sequence/_gen.py`, `diagram/STANDARD.md`
